# AgenC 0.8.3

A fix-only release: the TUI now runs production React, closing a memory
leak that could kill long sessions.

## Fixes

### Production React (long-session OOM)

- Production installs were running React in development mode: the React
  renderer stack is kept external to the bundle (one-singleton
  constraint), selects its dev/prod build from `NODE_ENV` at load time,
  and no install path ever set it. The development build's scheduling
  profiler records a user-timing entry for every unit of React work, and
  those entries are retained for the life of the process — so a long
  animated session (spinners running through an hour-long swarm turn)
  accumulated gigabytes of live heap until Node aborted the TUI with
  `FATAL ERROR: Reached heap limit` / SIGABRT.
- Process entries now pin `NODE_ENV` to `production` before any module
  can load (import-free wrapper entries; an explicitly set `NODE_ENV`
  still wins). The same workload that previously grew ~500MB in 15
  minutes now holds flat, and production React makes the TUI faster
  across the board.

## Upgrading

```sh
agenc update            # standalone installs
npm install -g @tetsuo-ai/agenc@latest   # npm installs
```

Restart the daemon after updating: `agenc daemon restart`.
