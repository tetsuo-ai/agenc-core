# AgenC 0.9.0

AgenC 0.9.0 hardens the live multi-agent surface into a correlated execution
protocol: conservative routing, atomic assignment, journal-first outcomes,
bounded context, and immutable Git evidence for isolated writers.

## Highlights

### `/swarm` is adaptive

- `/swarm on` keeps one agent as the default. It recommends parallel work only
  when the request explicitly asks for parallelism, or explicitly identifies
  independent items in a list.
- Qualifying work normally receives a two-worker recommendation, with a
  guidance ceiling of four. Coupled work and explicit no-delegation requests
  stay sequential; high-risk fan-out is capped at two.
- Routing considers only the exact runtime-owned root-human turn. The
  `agenc.swarm.route.v1` receipt includes a SHA-256 fingerprint of that input.
  Stale transcript text and synthetic completion turns are not treated as new
  swarm requests.
- Routing remains advisory. It does not spawn workers, create worktrees,
  approve tools, expand permissions, bypass budgets, or merge code.

### Assignments have an admission and an outcome

- `assign_task` atomically reserves one live `idle` descendant. Competing
  assignments cannot double-book the same worker.
- Every accepted assignment receives a `task_id` and fresh `turn_id`, then ends
  as `completed`, `errored`, `interrupted`, or `nack`.
- `nack` records work accepted but torn down before execution began.
- The child journal durably records `subagent_turn_outcome` before AgenC
  attempts parent-mailbox delivery. If that commit fails, the worker fails
  closed instead of presenting the result as durable.
- Parent delivery remains a separate bounded, live-process step, not a
  cross-journal exactly-once guarantee. Receipts include a stable
  `projection_id` and durable child-outcome reference.

### Isolated writers return Git evidence

- Each worktree spawn gets a spawn-unique path and branch, preventing a later
  logical respawn from inheriting retained state.
- After every isolated turn, AgenC captures the base commit, HEAD, tree hash,
  cleanliness, and ancestry, then rechecks HEAD and status for movement during
  capture.
- Only `committed_clean` work receives an `integration_ref`, set to the exact
  immutable Git commit ID. `unchanged_clean`, `dirty_uncommitted`, `diverged`,
  and `unverifiable` results do not.
- Dirty, diverged, or unverifiable workers are not returned to the reusable
  pool. A successful task outcome is not merge authorization, and AgenC never
  auto-merges.

### Coordination is bounded and authenticated

- Agent identity used by coordination tools is authenticated by the runtime
  instead of trusted from model-supplied session metadata.
- Root mailbox retention, daemon/TUI queued input, child passive context, and
  per-turn model projection have independent count and byte limits.
- Human input and task receipts are protected from passive chatter. Omitted
  passive traffic produces visible sequence-aware summaries.
- Passive `send_message` traffic does not start a provider turn. Inter-agent
  messages are limited to 65,536 characters and 65,536 UTF-8 bytes; larger
  messages are rejected.
- Failed pre-turn submissions roll back only their own attachment/skill batch
  and restore the composer, preventing hidden context from leaking into a later
  prompt.

### The 0.8.5 resume-card fix is complete

- This release completes the 0.8.5 resume-card repair for replayed
  `spawn_agent` and `close_agent` calls and streamed tool blocks. Those paths
  now register their tool names with the TUI renderer instead of producing
  “Tool use unavailable” cards.

### Release hardening

- Coordinator-mode guidance now tells the model to reuse an `idle` worker,
  matching the enforced `assign_task` admission contract.
- The release lock moves `shell-quote` to 1.10.0 and `fast-uri` to 3.1.4, and
  pins the MCP SDK's Hono adapter to 2.0.11. These are the patched lines for
  [CVE-2026-13311](https://github.com/advisories/GHSA-395f-4hp3-45gv),
  [CVE-2026-16221](https://github.com/advisories/GHSA-v2hh-gcrm-f6hx), and
  [GHSA-frvp-7c67-39w9](https://github.com/advisories/GHSA-frvp-7c67-39w9).

Full contracts: [agents reference](../reference/agents.md),
[`/swarm` reference](../reference/slash-commands.md#swarm), and
[swarm orchestration design](../design/swarm-orchestration.md).

## Compatibility notes

- `wait_agent` no longer accepts `target`. It drains all currently delivered
  mailbox updates and is classified as mutating. Use `list_agents` for a
  read-only status check.
- `assign_task` no longer queues work onto a running or `completed` worker. The
  sender must be a strict ancestor, the target must be a live `idle`
  descendant, and only one assignment may be outstanding.
- `completed` is terminal and non-reusable. The normal reusable-worker
  lifecycle is `pending_init → running → idle → running … → shutdown`.
- Worktree consumers must use the locators returned by AgenC. Path and branch
  names are spawn-unique and must not be derived from `task_name`. Integrate
  only the exact `base_commit..integration_ref` range returned with
  `committed_clean` evidence.
- Existing persisted swarm mode remains enabled after upgrade, but its behavior
  becomes the conservative adaptive policy described above.
- No project-state or config-file migration is required. The embedding SDK
  remains at 0.2.0.

## Research context

These are external results, not AgenC benchmark scores.

- Geng and Neubig’s CAID isolation ablation reports PaperBench Code-Dev scores
  of `55.5` with shared/soft isolation and `63.3` with worktree isolation
  (`57.2` single-agent) under that paper’s model, budget, and 20-paper setup:
  [arXiv:2603.21489v2](https://arxiv.org/abs/2603.21489v2).
- Kim et al. report that all evaluated multi-agent architectures underperformed
  the single-agent mean on sequential PlanCraft, by `39.1%` to `70.0%` in their
  matched-compute study:
  [arXiv:2512.08296v3](https://arxiv.org/abs/2512.08296v3).

Those results motivate worktree isolation and conservative, task-structure-aware
routing; they do not validate AgenC’s implementation. The
[design document](../design/swarm-orchestration.md#published-benchmarks-and-current-research-evidence-external)
records the full evidence set, limitations, and AgenC’s local evaluation gate.

## Upgrading

```sh
agenc update                              # standalone installs
npm install -g @tetsuo-ai/agenc@latest   # npm installs
```

Restart the daemon after updating: `agenc daemon restart`.
