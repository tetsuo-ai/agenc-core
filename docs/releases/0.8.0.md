# AgenC 0.8.0

AgenC 0.8.0 ships the verified-change workflow — the M5 milestone: one
command that takes an engineering goal to a review-ready, evidence-backed
patch, surviving daemon restarts and client disconnects without losing work
or repeating side effects.

## Highlights

### The verified-change workflow (`agenc run start`)

```sh
agenc run start --goal "fix the race in the retry counter" \
  --verify "unit=npm test" --follow
```

- Fixed pipeline: intake + policy check → isolated worktree → plan →
  implement → required verification → independent fresh-context review →
  sealed evidence bundle. One durable run ID across CLI, TUI, and SDK
  (`client.startRun`); `run status` shows the step table, `run evidence`
  exports the machine-readable bundle, `run cancel` cascades.
- Your checkout is never touched: the exact base commit and dirty state are
  recorded at intake, all changes happen in an isolated worktree, and the
  patch is exported and content-addressed before any cleanup. Base movement
  is detected and classified (clean 3-way vs explicit conflict), never
  overwritten.
- Completion is earned, not asserted: every required verification command
  must exit 0, an adversarial verification agent must return `VERDICT:
  PASS`, an independent reviewer — running in a fresh context that sees
  only the task, the diff, and the evidence — must report zero blockers,
  and the evidence record must validate mechanically against a sealed,
  hash-chained ledger. Anything else terminates with a machine-readable
  stop reason, and unresolved risks are reported honestly.
- Crash-safe by construction: steps are durable effects over the 0.7.2
  admission/durability kernel. Idempotent steps re-execute under
  content-derived keys; agent spawns are adopted from their durably
  recorded outcomes — never silently respawned. A daemon SIGKILL
  mid-implement resumes to a completed, verifier-passing patch.
- The exported evidence bundle is self-contained: an offline reader can
  re-derive and re-verify the whole run (hash chain, command records, patch
  digests, review output, terminal status) from the bundle alone, and a
  single tampered byte fails loudly.

### Grok streaming resilience

- The daemon stall watchdog now defers to the provider's stream-idle
  recovery, so healthy-but-paused Grok streams are no longer cancelled
  mid-thought.
- Streams that omit `output_text.delta` no longer render successful-but-
  blank turns; the completed-response text reaches the TUI.
- CommonMark soft line breaks render as spaces; one-shot agents always
  release daemon resources after their terminal snapshot; turns that end
  with reasoning but no visible output get one bounded retry.

### TUI

- The AskUserQuestion picker is redesigned and hardened: soft skip, ESC
  ownership, unified elicitation, and label-only options (the common Grok
  shape) are accepted.
- `/provider` shows xAI OAuth sign-in as a grok credential.

### Unknown-outcome recovery

- New `/resolve` command for reviewing and resolving `unknown_outcome`
  tool effects from the session, with timeouts honestly recorded as
  failures and tolerance for legacy-unsequenced effect records.

## Upgrading

```sh
agenc update            # standalone installs
npm install -g @tetsuo-ai/agenc@latest   # npm installs
```

Restart the daemon after updating: `agenc daemon restart`.
