# AgenC 0.8.1

A fix-and-hardening release on top of 0.8.0's verified-change workflow.
Includes a state schema upgrade (v16): the daemon migrates existing state
forward on first start.

## Fixes

### Authentication

- xAI OAuth refresh now survives concurrent refreshes across the TUI and
  daemon processes. A cross-process race could let one process quarantine a
  credential while another had just rotated it, flapping an active grok
  session to "Not logged in." Refresh is single-flight across processes,
  adopts a sibling's rotation instead of re-exchanging a stale token, and
  only reports "run /grok-login" when the grant is genuinely dead.

### Usage and rate display

- The token-ledger fallback no longer adds cached tokens (a subset of input)
  into the total, which had inflated displayed usage and cost.
- The spinner's tok/s is now measured over active streaming windows only —
  not wall-clock that includes tool execution — and no longer labels the
  model "slow" from silence while tools run. The rate is marked as an
  estimate.

### Long-session reliability

- Long-lived sessions are hardened: sandbox pseudo-devices, determinate
  failure classification, per-attempt step identities, and a 300s stall
  watchdog. State schema advances to v16.

### Verified-change workflow (M5)

- A crash during independent review now resumes by adopting the reviewer's
  durably recorded outcome instead of terminating `unknown_outcome`; a
  reviewer that genuinely died mid-flight still terminates honestly.
- Child usage is rolled up from reconciled admission usage; held-unknown
  reservations are surfaced, never counted as spend.

### TUI

- `esc` returns keyboard focus from the explorer/agents panes to the
  composer, and leaves the basic-fallback buffer with honest fallback hints;
  the persisted effort level is wired through.
- Compact-boundary helpers tolerate undefined message arrays instead of
  throwing.

### Performance

- Removed an interactive-session slowdown (snapshot-prune hot path and xAI
  store).

## Upgrading

```sh
agenc update            # standalone installs
npm install -g @tetsuo-ai/agenc@latest   # npm installs
```

Restart the daemon after updating: `agenc daemon restart`. The daemon
migrates existing state to schema v16 on first start.
