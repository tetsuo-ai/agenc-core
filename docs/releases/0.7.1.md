# AgenC 0.7.1 (in progress)

AgenC 0.7.1 turns the TUI into a full workbench for agent work — live
visibility into parallel agents, a self-updating todo board, and readable
approval surfaces — and hardens the turn lifecycle, tool schemas, and the
Linux sandbox underneath them.

This page tracks changes landed on `main` after 0.7.0 and is updated as the
release candidate settles.

## Highlights

### TUI workbench

- Project explorer with file-type icons and colors; open files with click or
  keyboard, scroll with mouse wheel or arrows, and inspect files without
  leaving the chat.
- `ctrl+r` review rail: moves the open file to a shiki-highlighted right-hand
  rail so you can review code while you keep chatting.
- Agents rail with a live swarm panel: per-agent progress, token counts, and
  durations while background agents run in parallel.
- Todo board shows itself below the composer whenever the agent has open
  tasks and hides again once everything completes. The board is shared with
  the daemon by conversation id, so what you see is what the agent writes.
- Chat polish: full-height colored gutters on messages, markdown headings
  rendered without raw `#`, clamped error cards, and real context-percent in
  the status line.

### Turn lifecycle hardening

- `esc` interrupts immediately in every state — including turns the daemon
  never acknowledged — and clears all local busy latches.
- Submit-ack watchdog (20s): if a submitted turn gets zero daemon response
  (e.g. an expired auth token), the UI recovers on its own and tells you,
  instead of spinning forever.
- Daemon-stall watchdog (60s): a turn that goes fully silent is marked
  interrupted and control returns to the composer; prompts queued behind it
  drain normally.
- Hung model streams no longer hold the busy state after the turn has ended.

### Tooling and approvals

- `AskUserQuestion` renders an interactive picker (numbered options, arrows,
  free-text Other) instead of a generic approval card, and the answers ride
  the `tool.approve` RPC back to the daemon. The schema accepts `preview` or
  `description` for option details.
- Plan approval overlay: the plan renders as clamped markdown (14 lines with
  `ctrl+o` to expand) with approve / review / keep-planning options always
  visible.
- `/effort` is back as a working command: show or set the reasoning effort
  level (`low`/`medium`/`high`) for the current model, validated against the
  model catalog. `grok-4.3` and `grok-4.5` supported.
- `spawn_agent`'s `message` field is documented in the tool schema so models
  stop omitting it.

### Linux sandbox

- The launcher honors the serialized `includePlatformDefaults` flag, so
  platform runtime directories are mounted whenever the policy asks for them.
- Bare program names (e.g. `rg`) now resolve against the command's PATH
  inside the namespace instead of dying with `execve ENOENT`.
- Sandboxed `Glob`/`Grep`/`exec` run end-to-end under restricted policies.

### Testing

- TUI suite aligned with the redesign contract: 842 files / 4,235 tests
  green, including new coverage for the plan-approval clamp, the todo-board
  store's fallback polling, and the AskUserQuestion parser.
