# AgenC 0.7.1

AgenC 0.7.1 makes agent work durable and budget-aware, delivers a complete TUI
workbench, and moves every active public install path onto one verified
five-platform runtime contract.

> This is the first complete public runtime and npm release after 0.6.1.
> Version 0.6.2 was an unpublished source candidate. The occupied
> `agenc-v0.7.0` tag is also source-only: it has no public runtime artifacts or
> npm package. Do not pin either version; install 0.7.1 or newer.

## Highlights

### TUI workbench

- The project explorer adds file-type icons and colors, mouse and keyboard
  navigation, and code inspection without leaving the conversation.
- `ctrl+r` moves an open file into a syntax-highlighted review rail so code and
  chat remain visible together.
- The agents rail shows live parallel-agent progress, token counts, and elapsed
  time.
- A daemon-backed todo board appears while work is open and hides when the
  plan completes, keeping the UI and agent on the same conversation state.
- Chat rendering now includes full-height message gutters, clean Markdown
  headings, bounded error cards, and an accurate context-usage indicator.

### Durable runs and safer effects

- Added `agenc run status`, `result`, `replay`, `evidence`, and `cancel`.
- Run events have durable IDs and per-run sequences, with explicit replay-gap
  reporting instead of silent loss.
- Terminal results remain inspectable after disconnects and daemon restarts.
- Tree-scoped cancellation stops queued and running descendants while
  preserving partial evidence.
- Side-effecting tool calls record intent before dispatch. Uncertain outcomes
  block dependent mutations until an operator reviews them with
  `agenc state resolve-tool-call`.

### Daemon-owned execution admission

- Model and charged-tool calls pass through one durable admission kernel.
- Hierarchical budgets, concurrency limits, durable reservations, exactly-once
  usage reconciliation, cancellation locks, and hashed evidence share one
  accounting authority.
- Heartbeat, cron, hooks, interactive sessions, and background agents use the
  same controls. Unknown usage remains reserved instead of being silently
  refunded.

### Verified installation and release integrity

- The v2 runtime manifest binds platform, architecture, native ABI, byte count,
  SHA-256, source commit, and compatibility floors.
- Official installs verify Sigstore/SLSA attestations against the expected
  source repository, workflow, commit, and release tag.
- Runtime installs are content-addressed, durably staged, and atomically
  activated. SQLite-backed locks and recovery journals serialize concurrent
  installs and wrapper updates.
- `agenc update`, the standalone macOS/Linux and Windows installers, and the
  npm launcher enforce the same artifact-safety contract.
- This release establishes the one-time legacy v1 manifest bridge for older
  updaters. Its digest and byte count are pinned after immutable publication
  and reused unchanged by later releases.
- Hosted release jobs build twice and publish only matching bytes for all five
  native targets: Linux x64/arm64, macOS x64/arm64, and Windows x64. Toolchain
  identity is recorded, and native build metadata is normalized so temporary
  checkout roots cannot change the runtime archive.

### Turn lifecycle, tools, and approvals

- `esc` interrupts immediately even when a submitted turn was never
  acknowledged, and clears local busy state.
- Submit-ack and daemon-stall watchdogs return control to the composer when a
  turn receives no response; queued prompts can then drain normally.
- Hung model streams no longer keep the UI busy after a turn has ended.
- `AskUserQuestion` has an interactive option picker with free-text fallback,
  and responses flow through the existing approval RPC.
- Plan approval renders bounded Markdown with approve, review, and
  keep-planning actions always visible.
- `/effort` can show or set supported reasoning effort for the current model.
- The `spawn_agent` schema now documents its required `message` field.

### Sandbox, security, and reliability

- Linux sandbox launches honor platform-default mounts and resolve bare
  commands such as `rg` against the in-namespace `PATH`.
- Sandboxed file search and command execution now run end to end under
  restricted policies.
- Repository content is treated as workspace guidance rather than trusted
  execution authority; unsafe aliases and stale blanket approvals fail closed.
- MCP mutations, sandbox process creation, subprocess ownership, and cleanup
  boundaries were hardened.
- MCP stdio transports ignore late child-close callbacks after process-tree
  teardown, preventing duplicate cleanup from crossing lifecycle boundaries.
- Large non-git workspace scanning no longer grows without bound. Heap-pressure
  warnings and near-OOM snapshots make memory failures diagnosable.
- Fixed onboarding races, embedded Neovim process leaks, buffer-provider
  transitions, prompt retry duplication, and opaque daemon startup failures.
- Expiring xAI OAuth sessions now refresh before admitted model calls, including
  xAI's expired-bearer `403` response path, instead of ending turns silently.
- Agent role identity is isolated by workspace.

### Evaluation tooling

- Added versioned evaluation suites, reproducible offline and real-model
  lanes, an allowlist egress proxy, batch baseline tooling, and a 7/7
  trust-conformance harness.
- Expanded regression coverage for TUI approval layouts, todo-state fallback
  polling, interactive questions, sandbox execution, installer verification,
  and native release reproducibility.

## Breaking changes and migration

- Node.js **`>=25.9 <26`** is required. Official artifacts use exactly Node.js
  `25.9.0`.
- Before the first 0.7.1 start, stop every AgenC process and back up the entire
  `$AGENC_HOME/projects` tree, including SQLite WAL files.
- Startup applies forward-only state migrations through schema v15. Older
  runtimes cannot open migrated databases; rollback requires restoring the
  complete pre-upgrade backup into a separate `AGENC_HOME`.
- `AGENC_HOME` must be an absolute path on a local filesystem. NFS, SMB,
  multi-host container volumes, and filesystems without the required locking
  and atomic-rename guarantees are rejected.
- `agenc budget reset` is rejected because durable accounting evidence cannot
  be erased. Use `agenc run status`, `evidence`, and `cancel`.
- Repository-provided MCP configuration and instruction includes may require
  renewed approval under digest-bound trust rules.
- Direct daemon `commandExec.start` and `thread/realtime/start` remain
  fail-closed until those paths have complete execution-admission coverage.

## Updating

Standalone installer users:

```bash
agenc update
agenc daemon restart
```

You can also rerun the installer or pin the release explicitly:

```bash
curl -fsSL https://get.agenc.ag/install.sh | sh -s -- --version 0.7.1
```

npm users:

```bash
npm install -g @tetsuo-ai/agenc@latest
agenc daemon restart
```
