# AgenC 0.7.0

AgenC 0.7.0 makes installation verifiable, execution budget-aware, and agent
work durable across disconnects, cancellation, and daemon restarts.

This release supersedes the unpublished 0.6.2 source candidate. GitHub-hosted
runner images changed before that candidate could produce any runtime
artifacts; no 0.6.2 runtime release or npm package was published. Do not pin
0.6.2.

## Highlights

### Durable runs and safer effects

- Added `agenc run status`, `result`, `replay`, `evidence`, and `cancel`.
- Run events now have durable IDs and per-run sequences, with explicit
  replay-gap reporting instead of silent loss.
- Terminal results remain inspectable after disconnects or daemon restarts.
- Tree-scoped cancellation stops queued and running descendants while
  preserving partial evidence.
- Side-effecting tool calls record intent before dispatch. Uncertain outcomes
  block dependent mutations until an operator reviews them with
  `agenc state resolve-tool-call`.

### Daemon-owned execution admission

- Model and charged-tool calls now pass through one durable admission kernel.
- Added hierarchical budgets, concurrency limits, durable reservations,
  exactly-once usage reconciliation, cancellation locks, and hashed evidence.
- Heartbeat, cron, hooks, interactive sessions, and background agents share the
  same accounting authority.
- Unknown usage remains reserved instead of being silently refunded.

### Verified installation and updates

- Introduced the v2 runtime manifest with exact platform, architecture, native
  ABI, byte count, and SHA-256 identities.
- Official installs verify Sigstore/SLSA attestations against the expected
  source repository, workflow, commit, and release tag.
- Runtime installations are content-addressed, staged durably, and activated
  atomically.
- Concurrent installs and wrapper updates are serialized with SQLite-backed
  locks and recovery journals.
- `agenc update`, the standalone installers, and the npm launcher now enforce
  the same compatibility and artifact-safety contract.

### Security and reliability

- Repository content is treated as workspace guidance rather than trusted
  execution authority.
- Project instructions are resolved securely on every turn; unsafe aliases and
  stale blanket approvals fail closed.
- MCP mutations, sandbox process creation, subprocess ownership, and cleanup
  boundaries were hardened.
- Fixed workspace scanning that could exhaust memory when AgenC was launched
  from a very large non-git directory.
- Added heap-pressure warnings and automatic near-OOM snapshots.
- Fixed onboarding races, embedded Neovim process leaks, buffer-provider
  transition races, prompt retry duplication, and opaque daemon startup
  failures.
- Agent role identity is now isolated by workspace.

### Evaluation tooling

- Added versioned evaluation suites, reproducible offline and real-model
  execution lanes, an allowlist egress proxy, batch baseline tooling, and a
  7/7 trust-conformance harness.

## Breaking changes and upgrade notes

- Node.js **`>=25.9 <26`** is required. Official artifacts use Node.js
  `25.9.0`.
- Stop all AgenC processes and back up the complete `$AGENC_HOME/projects`
  tree, including SQLite WAL files, before the first 0.7.0 start.
- Startup applies forward-only state schema migrations through v15. Older AgenC
  runtimes cannot open the migrated databases; rollback requires restoring the
  complete pre-upgrade backup into a separate `AGENC_HOME`.
- `AGENC_HOME` must be an absolute path on a local filesystem. NFS, SMB,
  multi-host container volumes, and filesystems without the required locking
  and atomic-rename guarantees are rejected.
- `agenc budget reset` is now rejected because durable accounting evidence
  cannot be erased. Use `agenc run status`, `evidence`, and `cancel`.
- Repository-provided MCP configuration and instruction includes may require
  renewed explicit approval under the new digest-bound trust rules.
- Direct daemon `commandExec.start` and `thread/realtime/start` currently fail
  closed until those paths have complete execution-admission coverage.

## Updating

Standalone installer users:

```bash
agenc update
agenc daemon restart
```

npm users:

```bash
npm install -g @tetsuo-ai/agenc@latest
agenc daemon restart
```
