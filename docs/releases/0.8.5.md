# AgenC 0.8.5

A fix-only release: two daemon-attached TUI display bugs.

## Fixes

### Assistant text no longer runs sentences together

- Consecutive assistant message segments rendered as one jammed
  paragraph ("…across subagents.No M1-named files…"): the live event
  wire carried per-token deltas but not the `agent_message`
  message-complete boundary that closes a streamed segment, so the TUI
  concatenated successive messages' deltas into one buffer with no
  separator. The boundary event now rides the live bridge, matching
  what rollouts always recorded.

### Resumed sessions render their tools again

- Resuming a session showed "Tool use unavailable" cards for any tool
  the session only used before the resume (most visibly
  `spawn_agent`/`close_agent` across resumed swarm sessions). The
  transcript replacement a resume delivers carries ready-made messages
  with no tool events, and the renderer's tool set was cleared without
  rescanning them. Replayed tool_use blocks now repopulate the render
  set.

## Upgrading

```sh
agenc update            # standalone installs
npm install -g @tetsuo-ai/agenc@latest   # npm installs
```

Restart the daemon after updating: `agenc daemon restart`.
