# AgenC 0.8.4

Grok streaming reliability — grounded in xAI's documented wire behavior —
plus real workbench metrics and a faster swarm.

## Fixes

### Grok: no more stream_idle reconnect loops

- xAI returns streamed function calls whole, in a single chunk (a
  documented WARNING in their function-calling docs): while the model
  generates a large tool-call payload, the wire carries zero bytes — no
  keepalives, no events. Measured: ~600 generated lines ≈ 118 seconds of
  total silence. Both watchdog layers previously read that silence as a
  stall, killed the healthy stream, and the retry regenerated the same
  file into the same timeout — the reconnect loop that made big Write
  calls with grok effectively impossible.
- Idle tolerance now follows xAI's own documented per-chunk idle default
  (600s): the session watchdog resolves env >
  `stream_watchdog_timeout_ms` config > provider-declared tolerance >
  default, and the grok adapter's streaming inter-chunk default rises
  from 120s to 600s. The `stream_watchdog_timeout_ms` config key is now
  actually wired (it previously did nothing).
- The grok adapter also maps every stream event it receives — streamed
  tool-call announcements render live in the TUI, raw reasoning deltas
  ride the thinking pipeline, and unmapped event types emit heartbeats so
  no future event type can starve the watchdog.

### Workbench metrics: real ctx%, honest tok/s

- ctx% was stuck at 0 forever: the daemon never forwarded per-stream
  usage to live clients. `token_count` (and streamed tool-argument
  events) now ride the live event bridge, and the context indicator
  reads the provider-reported usage.
- tok/s displayed ~3 on grok because the estimate read live buffers that
  reset every message/thinking block, and never counted tool-argument
  JSON. It is now cumulative across the turn (visible text + thinking +
  tool arguments).

## Performance

### Swarm

- Spawned workers fork clean from the task instead of the full parent
  history; each child gets its own admission parent-scope.
- Abandoned keep-alive workers are reaped to free registry slots; the
  per-agent summarizer skips no-op polls on a longer interval; subagent
  completions coalesce into one parent turn.

## Upgrading

```sh
agenc update            # standalone installs
npm install -g @tetsuo-ai/agenc@latest   # npm installs
```

Restart the daemon after updating: `agenc daemon restart`.
