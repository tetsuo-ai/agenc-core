# agenc-core read-only bug audit — 2026-07-13

Read-only audit (no code changed). Method: ~20 subsystem reviewers fanned out over
`runtime/src`, each finding then put through an independent adversarial refutation pass;
the highest-impact findings were additionally reproduced by executing the suspect code.
Priority order: (1) bugs/crashes, (2) error-handling gaps, (3) performance, (4) dead code.

Counts: 4 critical · ~30 moderate · ~30 minor · 3 refuted. Legend: ✅ = reproduced by
running the code; ✔ = confirmed by an adversarial verifier; the guard status of known-open
items is noted inline.

## CRITICAL

### C1. Secret sanitizer leaks Solana keypairs, PEM keys, the vault passphrase, and uppercase seed phrases ✅
`runtime/src/secrets/sanitizer.ts` (~270-351 patterns; 196 BIP39; 343-357/425-457 keys)
`redactSecrets()` returns these **unchanged** (executed against each): a Solana JSON-array
keypair (the standard `~/.config/solana/id.json` 64-int format), a `-----BEGIN PRIVATE KEY-----`
PEM block, `AGENC_WALLET_VAULT_PASSPHRASE=…` (the passphrase that unlocks the mainnet signing
wallet), and an ALL-CAPS BIP39 phrase (Ledger recovery sheets are uppercase). Causes: no
bracketed-int-array pattern, no PEM pattern, "passphrase" in no sensitive-key list, and
`classifyBip39Token` strips `[^a-z]` case-sensitively so `ABANDON`→miss. A `Read`/`cat` of a
wallet file, or a logged env/bash line, persists the signing key unredacted into `~/.agenc`
logs, rollout traces, and hook payloads — on a wallet-centric product.
FIX: add a 32/64-comma-separated-int-array pattern and a `-----BEGIN … PRIVATE KEY-----…END`
pattern; add `passphrase`/`credential` to the key lists; lowercase the token core before the
BIP39 wordlist lookup.

### C2. Read-only prefix disables the transaction guard entirely ✅
`runtime/src/transaction-guard/tool-intent.ts:120` (before 123)
`buildToolTransactionGuardInput` checks `isReadOnlySolanaLookup` (returns null = no guard)
BEFORE `hasTransactionWriteSignal`, and the read-only check only consults `SOLANA_WRITE_SIGNAL_RE`
— never `DIRECT_TRANSACTION_TOOL_RE`/`SOLANA_PROGRAM_WRITE_RE`, where `submitTransaction`,
`walletSign`, `write-buffer`, `set-buffer-authority` live (`\bsubmit\b`/`\bsign\b` miss
camelCase/hyphenated). Reproduced: `solana balance && node -e 'conn.submitTransaction(raw)'`
and `solana program show X && solana program write-buffer ./evil.so --url mainnet-beta` both
skip the guard; the same commands without the read-only prefix are guarded.
CAVEAT: the guard is **disabled by default** (`config.enabled` defaults false) — this defeats
an opt-in defense-in-depth layer, not a default-on control.
FIX: evaluate `hasTransactionWriteSignal` first; take the read-only early-exit only when it is
false. Add the camelCase/hyphenated terms to `SOLANA_WRITE_SIGNAL_RE`.

### C3. TOML config parser is vulnerable to `__proto__` prototype pollution ✅
`runtime/src/config/loader.ts` (`setNested` 386, `ensureTablePath` 427, `parseInlineTable` 340)
`parseToml` walks table/key paths with plain `cur[seg] = …` and never rejects `__proto__`.
Reproduced: after `parseToml("[__proto__]\nisAdmin = true\n")`, `({}).isAdmin === true` —
`Object.prototype` is poisoned for the whole daemon process, so every `obj.field ?? default`
config lookup across all concurrent sessions now sees the injected value. Reachable from any
config file the daemon reads (including a project-level `.agenc` config in a cloned repo).
FIX: reject/skip `__proto__`/`constructor`/`prototype` segments in `setNested`/`ensureTablePath`/
`parseInlineTable`, or build tables with `Object.create(null)`.

### C4. Responses-API transport replays unpaired tool items → session repeatedly 400s after an ESC-interrupt ✔
`runtime/src/services/api/openAiCodeTransform.ts:273` (`convertproviderMessagesToResponsesInput`)
Unlike the chat-completions path (`openaiShim.ts convertMessages` ~681-698/763-772, which drops
orphan tool blocks with a comment explaining that an ESC-interrupt creates a synthetic
`tool_result` with no recorded `tool_use`), this converter emits `function_call_output` for
every `tool_result` and `function_call` for every `tool_use` with NO pairing check. On the
gpt-5.x/Responses transport (`store:false`, full replay) an orphan persists in session history
and is replayed on every subsequent request → the session hard-fails repeatedly after an ESC.
NOTE: the unguarded emission + the orphan-producing scenario are verified; the exact provider
400 text is inferred.
FIX: apply the same pre-scan/pairing filter as `convertMessages`.

## MODERATE

### Cost / budget
- **Opus 4.5–4.8 mispriced 3× ✅** — `session/cost.ts:554` `canonicalModel()` collapses every
  `claude-opus-4*` to `claude-opus-4-7` @ `COST_TIER_OPUS` ($15/$75), but the canonical
  `utils/modelCost.ts` prices Opus 4.5–4.8 at `COST_TIER_5_25` ($5/$25). The live CostSidecar
  overcharges every Opus 4.5–4.8 turn 3×. FIX: give 4.5–4.8 their own $5/$25 entries; keep
  $15/$75 only for opus-4/4.1.
- **Background-agent `dollar_cap` trips at 1/3 the budget ✔** — `app-server/background-agent-runner.ts:2772`
  prices via the same stale `DEFAULT_MODEL_COSTS`, so an Opus 4.5–4.8 background agent halts at
  ~1/3 the configured spend. Same root fix as above.

### LLM / providers
- **`413` substring misclassifies transient 5xx as context-overflow ✔** — `llm/errors.ts:406`
  `CONTEXT_WINDOW_EXCEEDED_MESSAGE_RE` has a bare `413`, so a 500/503 whose body contains "413"
  (a request/trace id) maps to the non-retryable `LLMContextWindowExceededError` → drops history
  / fails the turn instead of retrying. FIX: anchor to `\b413\b` or drop it (numeric `status===413`
  check already exists).
- **Structured-output parse on a truncated Responses reply fails the turn ✔** —
  `llm/wire/responses-openai.ts:436` parses partial JSON when `finishReason==='length'` and throws
  "invalid JSON instead of a schema object" instead of surfacing recoverable truncation. FIX:
  gate the parse on a completed finishReason, mirroring the chat-completions parser.
- **Grok incremental-tracker leak ✔** — `llm/providers/grok/incremental.ts:234` registers each
  `GrokProvider` in a module-global Set removed only by `dispose()`, which no production path
  calls; auto-mode classification builds a fresh grok provider per classification → unbounded
  growth. FIX: wire provider teardown, or hold via WeakRef / drop the global registry.
- **OpenAI-shim writes `process.env` per query → cross-session contamination ✔** —
  `services/api/openaiShim.ts:2508` sets `OPENAI_BASE_URL`/`OPENAI_API_KEY`(=Gemini key). After
  any Gemini session runs, `isGeminiMode()` is true process-wide, changing routing/schema
  strictness for concurrent OpenAI-shim sessions. FIX: thread a local `providerOverride` instead
  of writing env.
- **Stale thinking-block index → RangeError ✔** — `services/api/openaiShim.ts:1257`
  `hasEmittedThinkingStart` is never reset, so a `reasoning_content` delta after the block closes
  emits `thinking_delta` at a stale index and the consumer throws (Kimi/MiniMax/Z.AI interleave
  reasoning around tool calls). FIX: open a fresh thinking block on a late reasoning delta.
- **Truncated tool call executed ✔** — `services/api/openAiCodeTransform.ts:667`
  `determineStopReason` returns `tool_use` before checking `incomplete_details==max_output_tokens`,
  so a tool call cut off mid-arguments is executed with JSON-repaired args. FIX: check
  `max_output_tokens` first.
- **Split tool-call id/name silently dropped ✔** — `services/api/openaiShim.ts:1303` registers a
  streaming tool call only when one delta has both `id` and `function.name`; providers that split
  them (vLLM/LM Studio/OpenRouter) lose the call and the agent loop stalls. FIX: register on `id`,
  patch name later, warn on orphan arg deltas.

### Daemon / concurrency
- **`listSessions`/`countSessions` are O(N²) under the session lock ✔** —
  `app-server/session-lifecycle.ts:218` scans the entire thread store per page inside the same
  AsyncLock that guards `getSession()` (hot message path), so paginating stalls all sessions.
  FIX: scan outside the lock (or cache) and push cursor/limit into `listThreads`.
- **Head-of-line blocking on the stdio connection ✔** — `app-server/transport/stdio.ts:168`
  serializes non-preemptive requests FIFO, and `message.stream`/`message.send` await the whole
  turn — so a concurrent control RPC on the same connection blocks for up to 30 min. FIX:
  dispatch full-turn streams off the per-connection FIFO (they have their own cancel path).

### Tools / exec
- **`Monitor` streams only the first 30s of a 30-min watch ✔** — `tools/system/monitor.ts:131`
  passes `yield_time_ms` = 30 min but `clampExecYield` caps it at 30s; nothing re-drives the
  process after, though the tool description promises ~1s polling. FIX: drive Monitor as a
  re-polling background task, or fix the description.
- **Unbounded subprocess buffering (shell mode) ✔** — `tools/system/bash.ts:347` accumulates all
  stdout/stderr into `Buffer[]` with no in-memory cap, truncating only at flush; a fast large
  emitter can OOM the daemon (all sessions). FIX: rolling byte cap during accumulation, matching
  `ProcessOutputBuffer.enforceCap`.
- **`enforceCap` O(cap) per append ✔** — `unified-exec/process-manager.ts:96` re-collapses the
  ~1MB pending region via slice/filter/join on every 8KB chunk when drain is deferred to end of
  yield; pins a core on the shared daemon. FIX: amortize/collapse lazily at drain time.
- **Background process evicted before it is polled ✔** — `unified-exec/process-manager.ts:396`
  `pruneExitedProcesses()` (run at the start of every `execCommand`) deletes any exited process,
  so a completed background task's final output + exit code are lost the moment the model issues
  the next exec — which is the normal start-task-then-poll workflow. FIX: don't prune an exited
  entry that was never drained.
- **Session/workspace read-state leaks forever ✔** — `tools/system/filesystem.ts:614`
  `sessionReadState` (≤25MB content per session) and `workspaceReadState` accumulate per closed
  session/path; the clear functions have zero production callers. FIX: wire `clearSessionReadState`
  into session close; evict `workspaceReadState`.
- **FileRead emits oversized images unresized ✔** — `tools/system/file-read.ts:1146` base64-encodes
  up to 10MB with no downsample/dimension clamp, but the provider limit is ~3.7MB/1568px and the
  BashTool/MCP paths resize; a large screenshot or high-DPI PNG is rejected/errors. FIX: route
  through `maybeResizeAndDownsampleImageBuffer`.

### Services / context
- **microcompact time-window protection is a no-op ✔** — `services/compact/microCompact.ts:91`
  relies on message timestamps, but `run-turn.ts:684/699` stamps every message `new Date(0)`, so
  the "don't clear results younger than 5 min" guard never fires and recent large tool results get
  stubbed out on the hot path. FIX: carry the real timestamp, or treat epoch-0 as unknown-age.

### TUI
- **Transcript re-mints all message UUIDs on every streaming delta ✔** —
  `tui/session-transcript.ts:430` synthetic messages use `randomUUID()`, and the transcript
  `useMemo` re-derives on every token, so every historical message gets a new key each delta →
  the whole virtual-list height cache is invalidated, all visible rows remount/re-measure, and
  cursor/selection/unseen-divider lose their target mid-stream. FIX: derive UUIDs deterministically
  from the source event key.
- **`ESC[3J` wipes terminal scrollback on resize/offscreen repaint ✔** — `tui/ink/clearTerminal.ts:68`
  every engine-internal full reset (resize, offscreen row change) emits `ESC[2J ESC[3J ESC[H`, and
  the `3J` erases the user's scrollback in the default main-screen mode. The deliberate ctrl+L path
  correctly uses only `2J`. FIX: use `2J`+home for engine resets; reserve `3J` for explicit clears.
- **`/context` fabricates a per-file breakdown ✅** — `tui/components/v2/ContextUsageModal.tsx:178`
  parses one aggregate `files: N` number but always renders three hardcoded filenames (`lib.rs`,
  `pool.rs`, `math.rs`) split by magic ratios — leftover fixture data shipped live; also
  `auto-compact at Infinity%` when hardLimit parses to 0. FIX: remove the fabricated rows or plumb
  real per-file data.
- **`Tabs` crashes on empty children ✔** — `tui/components/design-system/Tabs.tsx:235` indexes
  `tabs[selectedTabIndex][0]` unconditionally; a dynamically-built empty tab list throws on mount
  and `%0`→NaN. FIX: guard the empty case. (Reachability from a dynamic caller unproven.)
- **Neovim quit/close RPC rejection uncaught ✔** — `tui/workbench/buffer/neovim/NeovimLifecycle.ts:89`
  `isDirty()`/quit call `#rpc.request` with no catch; the transport closes independently of the
  session (EPIPE) so `:q`/close during that window escapes as an unhandled rejection (the sibling
  `#readCurrentDirtyState` wraps the same call). FIX: catch and treat a dead transport as
  not-dirty/closed.
- **MCP import dialog floats an async write ✔** — `tui/components/MCPServerDesktopImportDialog.tsx:114`
  `onSubmit` awaits `addMcpConfig` with no try/catch but `SelectMulti` invokes it fire-and-forget;
  an EACCES/EROFS write rejects as an unhandled rejection and the dialog hangs. FIX: try/catch the
  loop. (Same class: `PromptInput.tsx` submit path floats parent callbacks; SDK `subprocess.ts:259`
  writes stdin with no stream `error` listener → EPIPE can crash an embedder.)
- **AgentsRail arrow-nav follows a different order than it renders ✔** —
  `tui/workbench/agents/AgentsRail.tsx:24` navigates the flat task list but renders two partitioned
  sections, so ↑/↓ jumps between sections and can dispatch `taskId: undefined`. FIX: navigate the
  rendered (partitioned) order and guard the id.

### Onboarding (new code, commits 699768615 / 6c219902c — today)
- **Grok OAuth never shows the URL and swallows browser-open failure ✔** — `onboarding/Onboarding.tsx:168`
  `onAuthorizeUrl: (url) => { void openUrlInBrowser(url); }` — on a headless/SSH box the browser
  never opens, nothing is shown, and login blocks on the loopback callback for the full 300s with
  no feedback (the `/grok-login` command surfaces the URL; the wizard doesn't). FIX: display the

- **Theme "terminal background awareness" defaults to dark and can invert its advice ✔** —

### Permissions / agents
- **`bypassPermissions` waives user content-ASK rules ✔** — `permissions/bash.ts:471` under `--yolo`,
  `bashToolHasPermission` short-circuits to `allow` for any subcommand set with no explicit deny,
  so a configured `Bash(git push:*)`/`Bash(rm:*)` ASK guardrail is silently skipped (verified by
  running the function). FIX: before the `!hadDeny` early-return, surface any rule-based ask (reuse

- **Markdown agent roles resolve against the wrong workspace ✔** — `agents/v2/spawn.ts:511`

  multi-session daemon session A can spawn session B's same-named role. FIX: thread the requesting
  session's cwd into `requireAgentRole`/`getAgentRole`/`listAgentRoles` at every spawn/resume site.

### Bash read-only classification
- **`date --iso-8601 <MMDDhhmm>` misclassified as read-only ✔** — `tools/BashTool/readOnlyValidation.ts:740`
